# CipherVault v6.0 — Encrypted Backup + Browser Extension

## Phase 1: Encrypted Backup (v6.0) — Encrypted Export/Import

### Goal
Export the entire vault as a single encrypted `.ciphervault` file. Import it on any machine. No cloud, no server — just a file you can store on USB, send to yourself, or keep as backup.

### Architecture

**Export flow:**
1. User clicks "Export Encrypted Backup" in Settings
2. User sets a **backup password** (separate from master password)
3. Backend derives a key from backup password using PBKDF2 (600K iterations, new salt)
4. Backend reads the entire SQLite database as a buffer
5. Backend encrypts the buffer with AES-256-GCM using the derived key
6. Backend writes a `.ciphervault` file: `[16-byte salt][12-byte IV][16-byte authTag][encrypted DB]`

**Import flow:**
1. User selects a `.ciphervault` file
2. User enters the backup password
3. Backend derives key from password using stored salt
4. Backend decrypts and validates the buffer (AES-GCM will fail if wrong password)
5. Backend replaces the current vault.db with the decrypted database
6. App reloads

### File format: `.ciphervault`
```
[16 bytes] PBKDF2 salt (hex → binary)
[12 bytes] AES-256-GCM IV
[16 bytes] AES-256-GCM auth tag
[N bytes]  Encrypted SQLite database
```

### Implementation details

**backup.service.ts:**
```typescript
import { randomBytes } from 'crypto'
import { readFileSync, writeFileSync } from 'fs'
import { join } from 'path'
import { app, dialog, BrowserWindow } from 'electron'
import { getDatabasePath, saveDatabase, getDatabase } from '../db/connection'
import { deriveKey, splitDerivedKey, generateSalt } from '../crypto/keyderivation'
import { encrypt, decrypt } from '../crypto/encryption'
import { CRYPTO } from '../crypto/constants'

const MAGIC = 'CIPHERVAULT'  // 11 bytes magic header
const VERSION = 1

export async function exportEncryptedBackup(backupPassword: string): Promise<{ success: boolean; path?: string; error?: string }> {
  // 1. Ensure DB is saved
  saveDatabase()
  
  // 2. Read the entire database file
  const dbPath = getDatabasePath()
  const dbBuffer = readFileSync(dbPath)
  
  // 3. Derive key from backup password
  const salt = randomBytes(CRYPTO.SALT_SIZE)
  const key = deriveKey(backupPassword, salt)
  const { encryptionKey } = splitDerivedKey(key)
  
  // 4. Encrypt the database buffer
  const iv = randomBytes(CRYPTO.IV_SIZE)
  const cipher = createCipheriv(CRYPTO.ENCRYPTION_ALGO, encryptionKey, iv, {
    authTagLength: CRYPTO.AUTH_TAG_SIZE,
  })
  const encrypted = Buffer.concat([cipher.update(dbBuffer), cipher.final()])
  const authTag = cipher.getAuthTag()
  
  // 5. Show save dialog
  const win = BrowserWindow.getFocusedWindow()
  const result = await dialog.showSaveDialog(win!, {
    title: 'Export Encrypted Backup',
    defaultPath: 'cipher-vault-backup.ciphervault',
    filters: [{ name: 'CipherVault Backup', extensions: ['ciphervault'] }],
  })
  
  if (result.canceled || !result.filePath) return { success: false }
  
  // 6. Write file: magic + version + salt + iv + authTag + encrypted data
  const header = Buffer.concat([
    Buffer.from(MAGIC, 'ascii'),
    Buffer.from([VERSION]),
    salt,
    iv,
    authTag,
  ])
  
  writeFileSync(result.filePath, Buffer.concat([header, encrypted]))
  return { success: true, path: result.filePath }
}

export async function importEncryptedBackup(backupPassword: string, filePath?: string): Promise<{ success: boolean; error?: string }> {
  // 1. Show open dialog if no path
  if (!filePath) {
    const win = BrowserWindow.getFocusedWindow()
    const result = await dialog.showOpenDialog(win!, {
      title: 'Import Encrypted Backup',
      filters: [{ name: 'CipherVault Backup', extensions: ['ciphervault'] }],
      properties: ['openFile'],
    })
    if (result.canceled || !result.filePaths[0]) return { success: false }
    filePath = result.filePaths[0]
  }
  
  // 2. Read file
  const fileBuffer = readFileSync(filePath)
  
  // 3. Validate magic header
  const magic = fileBuffer.subarray(0, 11).toString('ascii')
  if (magic !== MAGIC) return { success: false, error: 'Invalid backup file' }
  
  const version = fileBuffer[11]
  if (version !== 1) return { success: false, error: 'Unsupported backup version' }
  
  // 4. Parse header
  const salt = fileBuffer.subarray(12, 44)    // 32 bytes
  const iv = fileBuffer.subarray(44, 56)       // 12 bytes
  const authTag = fileBuffer.subarray(56, 72)  // 16 bytes
  const encryptedData = fileBuffer.subarray(72)
  
  // 5. Derive key
  const key = deriveKey(backupPassword, salt)
  const { encryptionKey } = splitDerivedKey(key)
  
  // 6. Decrypt
  try {
    const decipher = createDecipheriv(CRYPTO.ENCRYPTION_ALGO, encryptionKey, iv, {
      authTagLength: CRYPTO.AUTH_TAG_SIZE,
    })
    decipher.setAuthTag(authTag)
    const decrypted = Buffer.concat([decipher.update(encryptedData), decipher.final()])
    
    // 7. Replace current database
    const dbPath = getDatabasePath()
    writeFileSync(dbPath, decrypted)
    
    return { success: true }
  } catch {
    return { success: false, error: 'Wrong password or corrupted file' }
  }
}
```

## Phase 2: Browser Extension (v7.0) — Chrome/Firefox Autofill

### Goal
A browser extension that communicates with the CipherVault Electron app to autofill passwords on web pages. No cloud — direct local connection via WebSocket.

### Architecture

**Components:**
1. **WebSocket server** — runs inside Electron app, listens on `ws://127.0.0.1:PORT`
2. **Browser extension** — Chrome/Firefox extension with content script for autofill
3. **Native messaging** — optional fallback for more secure communication

**Flow:**
1. User opens a website (e.g., github.com/login)
2. Content script detects login form fields (input[type=password], input[name*=user], etc.)
3. Content script sends `{ action: "search", domain: "github.com" }` to background script
4. Background script connects to Electron WebSocket server
5. Server searches entries matching the domain
6. Returns matching entries (encrypted — decrypted in Electron only)
7. Content script shows a small icon in the input field
8. User clicks icon → credentials autofilled

### Files to create/modify

**Browser extension (new directory: `extension/`):**
- `extension/manifest.json` — Manifest V3
- `extension/background.js` — WebSocket connection to Electron
- `extension/content.js` — Form detection + autofill
- `extension/content.css` — Autofill icon styling
- `extension/popup.html` + `popup.js` — Extension popup UI

**Backend:**
- `electron/main/services/websocket.service.ts` — NEW: WebSocket server
- `electron/main/index.ts` — Start/stop WebSocket server
- `electron/main/ipc/ipcHandlers.ts` — Add `extension:search-by-domain` handler

### WebSocket Protocol

**Extension → Electron:**
```json
{ "action": "search", "domain": "github.com" }
{ "action": "get-entry", "id": 42 }
{ "action": "generate-password", "options": { "length": 16 } }
```

**Electron → Extension:**
```json
{ "entries": [{ "id": 42, "title": "GitHub", "username": "user@email.com" }] }
{ "entry": { "id": 42, "username": "user@email.com", "password": "***" } }
```

### Security considerations
- WebSocket only on localhost (127.0.0.1) — no network exposure
- Extension must authenticate with the Electron app (shared secret generated at startup)
- Passwords never leave the Electron process — only username is sent to extension
- Extension requests password only when user explicitly clicks autofill
- Vault must be unlocked in Electron for extension to work
